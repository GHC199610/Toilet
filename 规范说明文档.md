## Core Runtime Facts

- 当前项目是单页前端应用。
- 主页面文件：`toilet_partition_auto_generator_V2Pro.html`。
- 后端绘图命令实验文件：`drawing-backend-v1.js`。
- PDF 依赖：`pdf-lib.min.js`。
- PWA 文件：`manifest.webmanifest`、`service-worker.js`。

## Units

- Business dimensions use millimeters (`mm`).
- Canvas/SVG coordinates use pixels after view-specific scaling.
- Area uses square meters (`m²`) when shown in cut lists.
- Keep user-facing dimensions consistent within one output. Do not mix `3000mm` and `3000 mm` in the same component unless an existing UI pattern requires it.

## Typography And Colors

Use existing CSS variables and established colors before adding new values.

Common drawing colors:

- Black `#000`: primary construction lines and dimensions.
- Blue `#185FA5`: emphasized door/label elements where already used.
- Red `#d64242` or established danger red: warnings/no-door markers.
- Light wall hatch/fill values must remain visually secondary.

## Data Flow Summary

```text
User input
→ normalized project/group/row model
→ validation and domain rules
→ calculation geometry
→ drawing/layout models
→ preview/export/print/cut list
```

Do not read rendered output to reconstruct business data.

## Naming Conventions

- User-facing cut-list name: “开料清单”.
- Drawing views: 平面图、正立面图、侧立面图、节点图.
- Door labels: group prefix plus booth number, e.g. `A1门`.
- Function names should follow existing project style; do not rename broadly during unrelated work.

## Verification Quick Reference

General code change:

- Inline script syntax check or equivalent.
- `git diff --check`.
- Target behavior check.

Drawing/view change:

- Canvas preview.
- PNG export when production route is affected.
- SVG/backend model check when backend route is affected.
- Non-target view check if shared helper changed.

Export/print/cut-list/mobile change:

- Follow `REGRESSION_PREVENTION_RULES.md`.

## Cleanup Rules

- Remove duplicate instructions instead of copying them into more files.
- Keep each spec focused on its owner domain.
- Avoid placeholder phrases such as “按现有规则”, “后续再说”, “预留”, unless the exact current rule is named.
- Prefer checkable rules over slogans.
