## 2. 组件联动规范

### 2.1 交互逻辑

#### 2.1.1 参数修改联动

| 修改参数 | 触发联动 | 联动方式 |
|----------|----------|----------|
| 间数 | 重新计算隔间宽度、尺寸分配 | 自动触发 `generate()` |
| 总长度 | 重新计算隔间宽度、尺寸分配 | 自动触发 `generate()` |
| 隔间深度 | 重新计算深度标注 | 自动触发 `generate()` |
| 门宽 | 重新计算尺寸分配 | 自动触发 `generate()` |
| 墙体条件 | 重新绘制墙体、立板 | 自动触发 `generate()` |

#### 2.1.2 视图切换联动

| 操作 | 联动效果 |
|------|----------|
| 切换标签页 | 重新绘制对应视图 |
| 缩放操作 | 保持当前视图缩放状态 |
| 重置视图 | 恢复默认缩放和位置 |

### 2.2 数据传递机制

```
用户输入 → getParams() → compute() → drawView() → 渲染Canvas
```

**数据流说明**：

1. `getParams()` 从 DOM 收集所有输入参数
2. `compute()` 根据参数计算派生值（隔间宽度、边缘、中间等）
3. `drawView()` 根据参数和计算结果绘制对应视图
4. `renderBOM()` 根据参数生成材料清单

### 2.3 状态同步规则

| 状态 | 存储位置 | 更新时机 |
|------|----------|----------|
| 当前参数 | `lastParams` | 每次点击生成时 |
| 当前视图 | `currentTab` | 切换标签时 |
| 缩放比例 | `zoomScale` | 滚轮操作时 |
| 平移位置 | `panX`, `panY` | 拖拽操作时 |
| 墙体模式 | `wallMode` | 选择墙体按钮时 |

### 2.4 事件触发与响应标准

| 事件 | 触发条件 | 响应动作 |
|------|----------|----------|
| `click` (生成按钮) | 用户点击 | 执行 `generate()` |
| `click` (标签页) | 用户点击 | 执行 `switchTab()` |
| `wheel` (Canvas) | 用户滚轮 | 更新缩放比例 |
| `mousedown` (Canvas) | 鼠标中键按下 | 开始拖拽 |
| `mousemove` (全局) | 拖拽中移动 | 更新平移位置 |
| `mouseup` (全局) | 鼠标中键释放 | 结束拖拽 |
| `resize` (窗口) | 窗口尺寸变化 | 重新调整 Canvas |

### 2.5 父子组件通信方式

本项目为单页应用，无传统父子组件结构。采用以下通信方式：

1. **DOM 数据绑定**：通过 `getElementById` 获取输入值
2. **全局状态变量**：通过 `lastParams`、`currentTab` 等全局变量共享状态
3. **函数调用**：直接调用 `generate()`、`drawView()` 等函数触发更新

## 3. 单位换算规范

### 3.1 长度单位使用标准

| 单位 | 使用场景 | 说明 |
|------|----------|------|
| `px` | Canvas 绘制 | 像素级精确控制 |
| `mm` | 业务数据 | 用户输入的实际尺寸 |
| `sc` (缩放因子) | 单位转换 | 将 mm 转换为 px |

### 3.2 尺寸换算公式

**基本公式**：
```
像素值 = 实际尺寸(mm) × 缩放因子(sc)
```

**缩放因子计算**：
```javascript
const scaleX = 可用宽度 / 总长度;
const scaleY = 可用高度 / 深度;
const sc = Math.min(scaleX, scaleY, 0.35); // 最大缩放不超过0.35
```

**示例**：
```javascript
// 实际尺寸 3000mm，缩放因子 0.1
// 像素值 = 3000 × 0.1 = 300px
const W = p.total * sc; // 3000 * 0.1 = 300
```

### 3.3 响应式断点定义

| 断点 | 宽度范围 | 适配策略 |
|------|----------|----------|
| 移动端 | < 768px | 单列布局，简化控件 |
| 平板端 | 768px - 1024px | 双列布局，保持完整功能 |
| 桌面端 | > 1024px | 完整布局，全部功能 |

### 3.4 数据单位格式化标准

| 数据类型 | 格式 | 示例 |
|----------|------|------|
| 长度 | `{数值}mm` | `3000mm` |
| 数量 | `{数值} 块` | `3 块` |
| 面积 | `{数值}m²` | `5.2m²` |
| 小数 | 保留1位小数 | `3.5` |
| 整数 | 四舍五入 | `1000` |

## 10. 数据处理规范

### 10.1 数据格式标准

| 数据项 | 类型 | 范围 | 默认值 |
|--------|------|------|--------|
| 间数 | number | 1-12 | 3 |
| 总长度 | number | 600-20000 | 3000 |
| 隔间深度 | number | 800-2000 | 1400 |
| 门宽 | number | 400-1000 | 600 |
| 立板高度 | number | 1000-2400 | 1830 |
| 门板高度 | number | 900-2400 | 1750 |
| 中隔板高度 | number | 1000-2400 | 1800 |
| 见光板高度 | number | 100-2400 | 1830 |
| 隔断板厚 | number | 6-25 | 12 |

### 10.2 数据验证规则

| 验证项 | 规则 |
|--------|------|
| 非空检查 | 输入不能为空 |
| 数值检查 | 必须为有效数字 |
| 范围检查 | 必须在规定范围内 |
| 逻辑检查 | 门宽不能超过隔间宽度 |

### 10.3 错误处理机制

```javascript
function getParams() {
  return {
    count: Math.max(1, parseInt(document.getElementById('p-count').value)||3),
    total: parseInt(document.getElementById('p-total').value)||3000,
    // 其他参数...
  };
}
```

**处理策略**：
1. 使用 `parseInt()` 转换，失败返回 `NaN`
2. 使用 `||` 提供默认值
3. 使用 `Math.max()` 确保最小值

### 10.4 数据缓存策略

| 数据 | 缓存方式 | 缓存时长 |
|------|----------|----------|
| 当前参数 | 全局变量 `lastParams` | 会话期间 |
| 缩放状态 | 全局变量 `zoomScale` | 会话期间 |
| 视图状态 | 全局变量 `currentTab` | 会话期间 |

### 10.5 接口请求/响应格式

本项目无外部接口，所有数据处理均在客户端完成。

## 附录：常用计算公式

### 尺寸分配公式

```
边缘 = (总长 - 门宽 × 间数) / (间数 × 2)
中间 = 边缘 × 2
```

### 隔间宽度公式

```
隔间宽度 = (总长 - (间数 + 1) × 板厚) / 间数
```

### 缩放因子公式

```
scaleX = 可用宽度 / 总长度
scaleY = 可用高度 / 深度
sc = Math.min(scaleX, scaleY, 0.35)
```
