# 回归事故防止规范

本文件用于约束后续所有代码修改，尤其是导出、打印、开料清单、移动端输入和图纸编辑相关功能。以下规则来自实际回归问题，必须作为强制检查清单执行。

## 0. 代码最高原则

- 单一事实源原则：同一份业务数据只能有一个源头。屏幕、导出、打印、计算不得各自维护一套“差不多”的逻辑。
- 入口矩阵原则：任何功能都不是一个按钮，而是一组入口。改一个入口前，必须列出同功能的所有入口。
- 端侧隔离原则：桌面端、移动端、横屏、竖屏不能互相假定。改移动端必须确认桌面端不变，反过来也一样。
- 渲染一致原则：屏幕看到什么，导出和打印默认就应该是什么。除非用户明确要求打印版另设样式。
- 降级一致原则：fallback 不能退回旧设计、旧数据、旧算法。兜底也必须输出当前版本结果。
- 最小影响原则：修 bug 只碰必要链路。公共函数能不动就不动，必须动就加条件边界。
- 不确定不承诺原则：没在真实入口跑过，就不能说“好了”“没问题”“手机端也可以”。
- 先复现再修复原则：用户说某入口坏了，必须先定位是哪条链路坏，不能直接猜改。
- 回归优先原则：修新问题前，先确认不会复活旧问题。尤其是导出、打印、移动端输入。
- 输出即生产原则：开料清单、图纸、板数属于生产资料，不能以“估计差不多”交付。
- 用户路径优先原则：按用户真实点击路径验证，不只调用底层函数验证。
- 状态顺序原则：选择状态、数据保存、重绘、聚焦、导出必须有明确顺序，不能靠碰巧。
- 可见边界原则：任何文字、标签、尺寸、按钮都不能超出容器；必要时自动缩放、换行或截断。
- 性能上限原则：高清、自动计算、导出都必须有上限，不能为了清晰度让手机崩溃。
- 提交隔离原则：只提交本次目标文件和规范文件，不带临时目录、不带无关改动。
- 失败显性原则：导出失败、打印被拦截、浏览器不支持时，必须给用户明确提示或可用 fallback，不能没反应。
- 命名一致原则：用户已经确定的业务名不能反复变，比如统一叫“开料清单”，不能又出现“材料清单”。
- 真实设备原则：移动端相关问题最终必须以手机真实表现为准，桌面模拟不能替代真实验证。

## 1. 总原则

- 不允许为了修一个端或一个入口，破坏另一个端或另一个入口。
- 修改公共函数前，必须先列出所有调用入口和输出链路。
- 未验证的内容不得说“没问题”“已完成”“不会影响”。
- 如果同一类问题重复出现，必须先复盘链路，再改代码，不允许继续靠猜测调参。
- 修改完成后必须说明：改了哪些入口、保护了哪些入口、哪些入口没有测试到。

## 2. 导出与打印矩阵

任何修改导出、打印、开料清单、图纸渲染时，必须同时核对以下矩阵：

- 桌面端：导出当前视图 PNG。
- 桌面端：导出全部视图 PNG。
- 桌面端：导出 PDF 当前视图。
- 桌面端：导出 PDF 全部视图。
- 桌面端：开料清单单独导出 PNG。
- 桌面端：开料清单单独导出 PDF。
- 移动端：导出当前视图 PNG。
- 移动端：导出全部视图 PNG。
- 移动端：导出 PDF 当前视图。
- 移动端：导出 PDF 全部视图。
- 移动端：开料清单单独导出 PNG。
- 移动端：开料清单单独导出 PDF。

SVG 入口必须单独核对，且必须遵守 `SVG_EXPORT_SPEC.md`：

- 桌面端：导出 SVG 当前视图。
- 桌面端：导出 SVG 全部视图。
- 桌面端：SVG 打印当前视图。
- 桌面端：SVG 打印全部视图。
- 桌面端：SVG 导出 PDF 当前视图。
- 桌面端：SVG 导出 PDF 全部视图。
- 移动端如保留 SVG 入口，也必须按同样矩阵核对。

SVG 是独立路线：`exportNamedSvg()` 与 `svgPrintPagesForScope()` 必须走 `renderBackendSheetSvg()`，不得切到 `renderSheetSvg()`、`renderSheetCanvas()`、PNG、普通 PDF 或普通直接打印路线。

如果入口有“包含开料清单”勾选项，必须额外验证：

- PNG 当前视图勾选开料清单时，不能只导出清单而丢失图纸。
- PNG 当前视图勾选开料清单时，图纸和开料清单必须同为 PNG 结果，不得混入 PDF/打印流程。
- PDF 勾选开料清单时，当前视图和全部视图都必须包含清单页。
- 平面图有多页时，当前视图 PNG/PDF 必须包含平面图所有分页，不得要求用户逐页导出。
- 全部视图必须包含平面图所有分页、正立面图、侧立面图。

## 3. 导出清晰度规则

- 图纸 PNG 和 PDF 必须使用高清渲染倍率，不得直接使用屏幕预览倍率。
- 修改清晰度时必须同时关注移动端内存，不得无限提高倍率。
- 图纸和开料清单合成一张 PNG 时，必须按目标画布宽度等比缩放，禁止出现清单变成底部小图的情况。
- PDF 打印页不得依赖浏览器默认缩放撑满，应使用明确的页面盒子和 `max-width/max-height` 适配。
- 不得让 `page-break-after: always` 在最后一页制造空白页。

## 4. 开料清单一致性规则

- 屏幕显示、PNG 导出、PDF 打印、图纸 PDF 附带清单必须使用同一套开料清单数据源。
- 不允许某个导出路径继续使用旧文本清单或旧 Canvas 画法。
- `前沿板` 可使用分组小卡片样式；`门板`、`中隔板`、`拼接板`、`见光板` 等普通项不得被误套前沿板样式。
- 普通项的工艺、高度、尺寸、数量必须和屏幕显示规则一致，不得导出时多显示、少显示或错位。
- PNG fallback 不得导出旧版内容。如果浏览器截图失败，fallback 也必须使用最新版结构。
- “材料清单”命名不得重新出现，统一使用“开料清单”。

## 5. 移动端输入与键盘规则

- 键盘弹出时，页面不得被顶到不可控位置。
- 用户编辑图纸文字、尺寸、标题、红色标签时，视图必须自动聚焦到正在编辑的位置。
- 编辑目标不得被键盘遮挡。
- 点击空白区域应能收起键盘，但不得误取消正在编辑的有效内容。
- 移动端修复不得破坏桌面端输入。
- 桌面端和移动端的编辑入口都必须验证：单击、双击、输入、回车、失焦。

## 6. 图纸文字编辑规则

- 标题输入框内容严禁超出标题框和图纸边界。
- 标题最多两行；内容过长时必须自动缩小字号。
- 缩小到最小字号仍放不下时，必须省略号截断，不能画出框外。
- 红色标签必须能在桌面端和移动端输入。
- 红色标签回车应确认并收起键盘；标题回车应支持换行，超过允许行数后应确认或阻止继续换行。
- 编辑框定位必须基于最新一次图纸重绘后的命中区域，不得使用旧 hit area。

## 7. 画布命中与重绘顺序规则

- 修改图纸对象选择、标签编辑、标题编辑时，必须检查 `selectionScope`、命中区域、重绘、聚焦的顺序。
- 需要定位输入框时，必须先完成图纸重绘，再根据最新命中区域定位输入框。
- 不得只修移动端定位而让桌面端输入框停留在错误位置。
- 双击逻辑不得只支持尺寸编辑；标题和红色标签也必须保持可编辑入口。
- 新增事件拦截时，必须只拦截目标对象，禁止全局吞掉非目标双击/点击。

## 8. 分页与图纸比例规则

- 平面图分页由图纸分页逻辑决定，导出不得私自改变分页数量。
- 当前视图如果是平面图，导出必须包含平面图所有分页。
- 当前视图如果是正立面或侧立面，只导出当前图种。
- PDF 应一页一图；PNG 可按当前产品逻辑合成一张图，但必须清晰且比例合理。
- 移动端打印必须按图纸实际分页输出，不得强行压成一页，也不得额外产生空白页。

## 9. 修改前必查链路

改任何相关功能前，必须先搜索并阅读以下函数或等价入口：

- 图纸 PNG：`exportNamedImage`、`exportNamedAllImages`、`downloadCanvasPng`。
- 图纸 SVG：必须遵守 SVG_EXPORT_SPEC.md；SVG 不得作为普通 PDF/打印/PNG 的中间格式，普通 PDF/打印/PNG 也不得作为 SVG 的替代路线。
- SVG 独立路线：`exportNamedSvg()`、`svgPrintPagesForScope()`、`directSvgPrint()`、`exportSvgPdfViaPrint()`、`renderBackendSheetSvg()`、backend drawing model builders、`drawing-backend-v1.js`。
- 图纸 PDF：`exportNamedPdf`、`exportPdfCanvasesForScope`、打印 iframe/window 触发函数。
- 开料清单：`renderBomItems`、`formatBomLines`、`formatBomMobileLines`、`bomPrintCardsHtml`、`bomPrintDocumentHtml`、`renderBomCanvas`、`exportBomImage`、`exportBomPrint`。
- 图纸绘制：`renderSheetCanvas`、`renderPlanSheetsStripCanvas`、分页相关函数。
- 图纸文字编辑：`focusCanvasTextEditor`、`positionCanvasTextEditor`、`activeCanvasTextHitArea`、`canvas-text-editor` 事件。
- 尺寸编辑：`startDimensionEdit`、`dimension-editor` 事件。

## 10. 修改后最低验证

每次相关修改后至少执行：

- JS 语法检查。
- 目标入口验证。
- 同类非目标入口验证。
- 桌面端和移动端各至少一个关键路径验证。
- 如果没有真实浏览器验证，必须明确说明“未做真实浏览器点击验证”。

禁止只看代码后直接断言桌面端或移动端“都好了”。

## 11. 交付说明要求

完成后必须明确说明：

- 本次改动影响哪些导出/输入入口。
- 本次没有改动哪些入口。
- 实际跑过哪些检查。
- 哪些需要用户刷新后在真实设备验证。
- 是否已提交和推送。

## 12. 禁止行为

- 禁止把 PNG 导出修成 PDF 打印。
- 禁止让勾选开料清单后只导出清单、不导出图纸。
- 禁止让清单 PNG 内容和屏幕清单不一致。
- 禁止移动端修复造成桌面端输入失效。
- 禁止修改公共导出函数后不检查所有导出入口。
- 禁止为了当前截图正确而扩大公共函数行为范围。
- 禁止未验证就承诺“手机端没问题”。
- 禁止把 SVG 导出、SVG 打印、SVG 导出 PDF 改成 Canvas、PNG、普通 PDF、普通打印或 `renderSheetSvg()` 路线。
- 禁止把普通 PNG/PDF/直接打印改成 backend SVG 路线。
- 禁止在未明确获得用户批准前改变 SVG 独立架构。
